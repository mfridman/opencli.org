open cli - draft

10,000 foot overview:

My pipe dream is if all these frameworks outputted a parsable representation of the CLI.

Most CLIs nowadays are written in a particular fashion, whereby the sequence of commands is represented by a directed rooted tree composed of at least:

  1. Root command
  2. Subcommands
  3. Short and long command descriptions
  4. Global flags and subcommand-specific flags
  5. Short and/or long flag descriptions
  6. Environment variables

From what I can tell, this is the absolute minimum required to describe a CLI that covers the most amount of language frameworks.
